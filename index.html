Séance 5 sur 24 — Acte 2

Ton étude de marché.
Avant de construire, tu comprends.

Bienvenue dans l'Acte 2. À partir de maintenant, tu construis ton business. Et la première chose à faire — avant le site, avant le blog, avant LinkedIn — c'est de comprendre ton marché. Qui sont tes concurrents, ce qu'ils vendent, à quel prix, et où sont les trous. C'est ce qu'on fait dans cette séance.

40-50 min

Pourquoi c'est la première séance de l'Acte 2.

Parce que si tu construis ton site, ton blog, ton tunnel sans savoir à qui tu parles, ce que les autres font, et où sont les opportunités — tu construis dans le vide.

Tout ce que tu fais après dépend de cette séance : ton positionnement (séance 7), ton site (séance 8), ton blog (séance 9), tes posts LinkedIn (séance 12), ta prospection (séance 15). Si ton étude de marché est solide, tout le reste tombe en place.

Ce que tu vas avoir à la fin de cette séance

→ Un tableau comparatif de tes 10 concurrents (offres, prix, forces, faiblesses)

→ Une analyse de ce que les gens cherchent vraiment

→ Un benchmark des plateformes freelance

→ Ton profil client idéal (secteur, taille, décideur, problème, budget)

→ Les trous dans le marché — ce que personne ne fait

→ Ton positionnement, tes prix, tes mots-clés

→ Le tout dans des fichiers HTML visuels que tu gardes

Comment ça va se passer.

7 phases. À chaque phase, tu copies un prompt, tu le personnalises avec tes infos, tu le colles dans Claude Code, et il fait le boulot. À chaque phase, Claude Code enrichit le même fichier : etude-marche.md. À la fin, phase 7, ChatGPT génère un HTML visuel et classe.

C'est une recette. Tu suis les étapes, tu ajustes les ingrédients (tes infos), et tu obtiens ton étude de marché.

0

D'abord, tes infos.

Avant de lancer quoi que ce soit, tu dois personnaliser les prompts. Remplace les variables entre crochets [...] par tes infos. C'est ce qui fait que l'étude sera adaptée à TON projet, pas un truc générique.

Tes variables

Prends 2 minutes pour répondre à ces questions. Tu vas réutiliser ces réponses dans chaque prompt.

[TON_PROJET]

Freelance expert IA ? Agence d'automatisation ? Consultant IA ? Formateur IA ?

Exemple : "freelance expert IA spécialisé en automatisation pour PME"

[TES_SERVICES]

Qu'est-ce que tu veux vendre ? Audit IA, chatbots, automatisation CRM, agents vocaux... ?

Exemple : "audit IA, chatbot SAV, assistant email, agent vocal, automatisation CRM"

[TA_CIBLE]

Qui tu veux comme clients ? Restaurants, cabinets comptables, agences marketing, e-commerce... ?

Exemple : "TPE/PME de 5-50 salariés dans les services"

[TA_ZONE]

Tu cibles où ? Une ville, une région, la France entière, international ?

Exemple : "Lyon et sa région" ou "France entière"

[TON_BUDGET_CIBLE]

Combien tu veux facturer ? Quelle fourchette de prix ?

Exemple : "1 500 à 8 000€ par projet"

Si tu sais pas encore

C'est normal. Mets ce qui te parle le plus pour l'instant. L'étude de marché va justement t'aider à affiner. Tu pourras toujours relancer les prompts avec des variables différentes après.

1

Phase 1 — Analyser les concurrents.

Claude Code va naviguer sur Google, trouver tes 10 principaux concurrents, visiter leurs sites un par un, et extraire tout ce qui est utile.

Le prompt — copie et personnalise

Je veux que tu fasses une analyse concurrentielle complète pour mon projet : [TON_PROJET]. Je cible [TA_CIBLE] dans la zone [TA_ZONE].

Étape 1 : Cherche sur Google les 10 premiers résultats pour ces requêtes :
- "[TON_PROJET] [TA_ZONE]"
- "consultant IA PME [TA_ZONE]"
- "automatisation IA entreprise [TA_ZONE]"
- "expert IA freelance [TA_ZONE]"

Étape 2 : Pour chaque concurrent trouvé, visite leur site et extrais :
- Nom et URL du site
- Leurs services (liste complète)
- Leurs prix (si affichés)
- Leur positionnement (comment ils se présentent en 1 phrase)
- Leur note Google et nombre d'avis
- Leurs forces (ce qu'ils font bien)
- Leurs faiblesses (ce qui manque, ce qui est mal fait)
- Présence LinkedIn (nombre d'abonnés, fréquence de publication)

Étape 3 : Génère un fichier HTML visuel et propre avec :
- Un tableau comparatif de tous les concurrents
- Un résumé des tendances (ce que tout le monde fait, ce que personne ne fait)
- Les opportunités identifiées
Crée un fichier etude-marche.md et écris dedans une section "## 1. Analyse concurrentielle" avec le tableau + résumé.

Exemple avec les variables remplies

// Exemple pour un freelance IA à Lyon
[TON_PROJET] → freelance expert IA spécialisé en automatisation pour PME
[TA_CIBLE] → TPE/PME de 5-50 salariés dans les services
[TA_ZONE] → Lyon et sa région

Requêtes Google :
→ "freelance expert IA Lyon"
→ "consultant IA PME Lyon"
→ "automatisation IA entreprise Lyon"
→ "expert IA freelance Lyon"

Ce que Claude Code fait

1

Il lance les recherches Google

2

Il visite chaque site concurrent avec Playwright (automatisation de navigateur)

3

Il extrait les infos de chaque page (services, prix, positionnement)

4

Il cherche les avis Google de chaque concurrent

5

Il compile tout dans un HTML propre avec un tableau et des insights

Toi tu fais rien. Tu copies le prompt, tu attends, tu ouvres le fichier HTML.

Le résultat

Claude Code crée etude-marche.md avec la section "1. Analyse concurrentielle". Le fichier va grossir à chaque phase. Tu l'ouvres dans ton navigateur, tu vois tous tes concurrents côte à côte : ce qu'ils font, à quel prix, leurs forces et leurs faiblesses. C'est la base de tout.

2

Phase 2 — Analyser la demande.

Maintenant que tu sais ce que les concurrents proposent, on regarde ce que les gens cherchent. Quels mots-clés ils tapent, quels problèmes ils ont, qu'est-ce qu'ils trouvent pas.

Le prompt — copie et personnalise

Je veux analyser la demande réelle pour mon projet : [TON_PROJET]. Je cible [TA_CIBLE] dans [TA_ZONE].

Étape 1 — Mots-clés Google : Cherche les mots-clés que les gens tapent sur Google en rapport avec [TES_SERVICES]. Pour chaque mot-clé, estime le volume de recherche et la concurrence.

Étape 2 — Questions fréquentes : Cherche sur Google "People Also Ask", Reddit, forums, et Quora les questions que les dirigeants de [TA_CIBLE] posent sur l'IA et l'automatisation.

Étape 3 — LinkedIn : Cherche les posts LinkedIn les plus engagés sur les sujets "IA pour PME", "automatisation entreprise", "[TES_SERVICES]". Note les sujets qui marchent et ceux qui marchent pas.

Étape 4 : Génère un fichier HTML avec :
- Top 20 mots-clés (volume, concurrence, opportunité)
- Top 10 questions que les gens posent
- Top 5 sujets LinkedIn qui marchent
- Les "trous" : ce que les gens cherchent et ne trouvent pas
Enrichis etude-marche.md avec la section "## 2. Analyse de la demande".

Le résultat

Tu sais maintenant ce que les gens cherchent. Ces mots-clés tu les réutilises pour ton blog SEO (séance 9) et tes posts LinkedIn (séance 12). Les questions fréquentes deviennent tes contenus. Les trous deviennent ton positionnement.

3

Phase 3 — Benchmark plateformes.

On regarde ce qui se vend vraiment sur les plateformes freelance. Pas ce que les concurrents disent qu'ils font — ce qui se vend pour de vrai, à quel prix, avec quels avis.

Le prompt

Je veux analyser ce qui se vend sur les plateformes freelance pour [TES_SERVICES].

Étape 1 — Malt : Cherche les profils qui proposent des services similaires à [TES_SERVICES] sur malt.fr. Pour les 10 premiers : tarif journalier, services proposés, note, nombre de missions, ce qui manque dans leur offre.

Étape 2 — Upwork : Même chose sur upwork.com pour les freelances qui vendent [TES_SERVICES]. Tarifs, services, avis, forces, faiblesses.

Étape 3 — Fiverr : Pareil sur fiverr.com. Note les prix les plus bas et les plus hauts pour des services similaires.

Étape 4 : Génère un fichier HTML avec :
- Tableau comparatif des 3 plateformes (nombre de freelances, tarifs moyens, services les plus vendus)
- Les services les plus demandés vs les moins proposés
- Les fourchettes de prix du marché
- Les opportunités (ce qui manque, ce qui est mal fait)
Enrichis etude-marche.md avec "## 3. Benchmark plateformes".

Le résultat

Tu vois les vrais prix du marché. Pas les prix que les concurrents affichent sur leur site — les prix que les gens paient vraiment. C'est ça qui va t'aider à fixer tes tarifs.

4

Phase 4 — Ton client idéal.

Maintenant que tu connais le marché (concurrents + demande + plateformes), on définit précisément qui tu vises.

Le prompt

À partir de l'analyse des concurrents (01-concurrents.html), de la demande (02-demande.html) et des plateformes (03-plateformes.html), définis mon profil client idéal.

Mon projet : [TON_PROJET]
Mes services : [TES_SERVICES]
Ma zone : [TA_ZONE]
Mon budget cible : [TON_BUDGET_CIBLE]

Pour le profil client idéal, détaille :
- Secteur d'activité (2-3 secteurs prioritaires avec justification)
- Taille de l'entreprise (nombre de salariés, CA approximatif)
- Le décideur (qui prend la décision d'achat : gérant, DG, DAF, DSI...)
- Ses 3 plus gros problèmes que l'IA peut résoudre
- Son budget probable pour des services IA
- Où le trouver (LinkedIn, Google, événements, réseaux...)
- Ce qui le convaincrait de bosser avec toi (arguments clés)

Génère un fichier HTML visuel avec le profil complet, illustré et structuré.
Enrichis etude-marche.md avec "## 4. Profil client idéal".

Le résultat

Tu sais exactement qui tu cibles. Quand tu crées ton site (séance 8), tu parles à cette personne. Quand tu prospectes (séance 15), tu cherches cette personne. Plus de "je vends à tout le monde" — tu as une cible précise.

5

Phase 5 — Les trous dans le marché.

La phase la plus stratégique. On croise toutes les données pour trouver ce que personne ne fait — et c'est là que tu te positionnes.

Le prompt

À partir de toutes les analyses précédentes (01-concurrents.html, 02-demande.html, 03-plateformes.html, 04-client-ideal.html), identifie les trous dans le marché.

Mon projet : [TON_PROJET]
Mes services : [TES_SERVICES]

Analyse :
1. Ce que TOUS les concurrents font → c'est le standard, je dois le proposer aussi
2. Ce que les concurrents font MAL → c'est l'opportunité d'être meilleur
3. Ce que PERSONNE ne fait → c'est mon positionnement différenciant
4. Ce que les clients DEMANDENT mais ne TROUVENT PAS → c'est l'opportunité la plus grande
5. Les segments de clientèle IGNORÉS par les concurrents → c'est ma niche potentielle

Pour chaque trou identifié, propose :
- Le service que je pourrais créer
- Le prix que je pourrais facturer
- L'argument de vente en 1 phrase

Génère un fichier HTML visuel avec les trous classés par opportunité (du plus grand au plus petit).
Enrichis etude-marche.md avec "## 5. Trous dans le marché".

Le résultat

Tu vois noir sur blanc où sont les opportunités. "Tous les consultants IA vendent de la formation. Personne ne connecte l'IA aux outils existants du client." Boom — c'est ton angle.

6

Phase 6 — Ton positionnement final.

On compile tout. À la fin de cette phase, tu as ton positionnement, tes prix, tes messages, et tes mots-clés. Tu sais exactement qui tu es sur le marché.

Le prompt

À partir de toutes les analyses (01 à 05), crée mon document de positionnement final.

Mon projet : [TON_PROJET]
Mes services : [TES_SERVICES]
Ma cible : [TA_CIBLE]
Ma zone : [TA_ZONE]
Mon budget cible : [TON_BUDGET_CIBLE]

Le document doit contenir :

1. MON POSITIONNEMENT
- Ma phrase de positionnement (1 phrase qui résume ce que je fais et pour qui)
- Ce qui me différencie des concurrents (3 points max)
- Mon "pourquoi moi" en 3 phrases

2. MON OFFRE
- La liste de mes services avec description courte
- Ma grille tarifaire (prix basés sur le benchmark du marché)
- Mon service d'entrée (le plus facile à vendre)
- Mon service premium (le plus rentable)

3. MA CIBLE
- Résumé du profil client idéal
- Les 3 problèmes que je résous
- Les arguments qui convertissent

4. MES MOTS-CLÉS
- Top 10 mots-clés pour le SEO (blog, séance 9)
- Top 5 sujets pour LinkedIn (posts, séance 12)
- Top 3 angles pour la prospection (messages, séance 15)

5. MON PLAN D'ACTION
- Les 3 premières actions à faire cette semaine

Génère un fichier HTML complet, visuel, professionnel. C'est mon document de référence pour toute la suite de la formation.
Enrichis etude-marche.md avec "## 6. Positionnement final".

Quelques conseils.

Lance les phases une par une

Fais pas tout d'un coup. Lance la phase 1, vérifie le résultat, ajuste si besoin, puis passe à la phase 2. Chaque phase utilise les résultats de la précédente.

Vérifie les résultats

Claude Code peut se tromper sur des détails (un prix, un nombre d'avis). Ouvre les HTML, relis, vérifie les chiffres qui te semblent bizarres. C'est ton étude — elle doit être fiable.

Personnalise les prompts

Les prompts sont des bases. Rajoute des détails si tu veux. "Analyse aussi leurs pages LinkedIn en détail" ou "Regarde aussi les avis Trustpilot". Plus tu donnes de contexte, meilleur est le résultat.

Tu peux relancer

Si un résultat te plaît pas, relance le prompt avec plus de détails ou une direction différente. C'est pas gravé dans le marbre — itère jusqu'à ce que t'es satisfait.

Garde tes 6 fichiers

Mets-les dans un dossier "Étude de marché" sur ton Drive. Tu vas les rouvrir régulièrement pendant toute la formation. C'est ta boussole.

Les outils utilisés.

Claude Code → c'est lui qui fait tout. Il lance les recherches, navigue les sites, extrait les données, compile les résultats, génère les HTML.

Playwright → Claude Code utilise Playwright (automatisation de navigateur) pour visiter les sites des concurrents et extraire les infos. Tu installes rien — Claude Code gère.

Chrome / Web → Claude Code accède au web pour les recherches Google, les profils LinkedIn, les plateformes freelance.

HTML → les résultats sont générés en HTML. Des fichiers propres, visuels, que tu ouvres dans ton navigateur.

Les limites à connaître.

Le scraping peut être bloqué

Certains sites bloquent le scraping automatique. Si Claude Code arrive pas à accéder à un site, c'est normal. Il te le dira et passera au suivant.

Les prix sont pas toujours affichés

Beaucoup de concurrents affichent pas leurs prix sur leur site. Claude Code mettra "prix non affiché" — c'est une info en soi (ça veut dire qu'ils veulent qu'on les contacte).

Les volumes de recherche sont des estimations

Claude Code donne des estimations de volume de recherche, pas des chiffres exacts. C'est suffisant pour se positionner, mais c'est pas Google Keyword Planner.
